Understanding WMI/CIM

What is WMI/CIM?

WMI (Windows Management Instrumentation) and CIM (Common Information Model) are Windows management APIs that allow remote access to system information and control.

Term Description
WMI Microsoft's legacy implementation of CIM standards
CIM Modern, industry-standard approach (PowerShell uses this)
CimSession PowerShell's modern remote connection via WS-Man protocol

Your script uses New-CimSession which communicates via WinRM.

Understanding WinRM

What is WinRM?

WinRM (Windows Remote Management) is Microsoft's WS-Management protocol implementation for remote hardware management.

Property Value
HTTP Port TCP 5985
HTTPS Port TCP 5986
Purpose Remote PowerShell, WMI/CIM queries, system management
Default State Disabled on most Windows installations

Understanding "Access Errors: 334"

What This Error Means

The error "Access Errors: 334" indicates that your script successfully reached 334 computers (ping worked), but WMI/CIM queries failed due to permission or configuration issues.

Error Type Explanation
Online but Inaccessible 334 computers returned "WMI access denied or unavailable"
Root Cause Your account lacks admin privileges or WinRM is disabled on target machines

Common Causes

Missing admin rights Your account doesn't have admin privileges on 334 target machines
WinRM disabled Windows Remote Management service isn't running on those computers
Firewall blocking Windows Firewall blocks WMI/CIM traffic (TCP 135 + dynamic ports)
UAC restrictions Remote UAC is blocking administrative access
WinRM not configured PowerShell remoting isn't set up for those machines

Is This Normal?

Yes! Access errors between 20-50%% are normal in large enterprises where:

  • Laptops are off-site/VPN-only
  • Workstations are locked down for security
  • Security policies restrict WMI access

The script is working correctly - it's accurately reporting which machines are accessible for WMI auditing.

Best Solution: Group Policy (Recommended)

✓ Why Group Policy is Best

  • Scalable for enterprise deployments
  • Secure - no over-privileged accounts needed
  • Auditable and reversible
  • Industry best practice for Windows management
  • Automatic deployment to all GOT* computers

Step 1: Configure WinRM Service GPO

Path: Computer Configuration → Administrative Templates → Windows Components → Windows Remote Management (WinRM)

# WinRM Service Settings: # 1. Allow remote server management through WinRM: Enabled # 2. Trusted Hosts: Add your management server IP or configure Kerberos # 3. Allow Basic authentication: Disabled (use Kerberos)

Step 2: Configure Windows Firewall Rules

Path: Computer Configuration → Windows Settings → Security Settings → Windows Firewall

# Inbound Rules to Create: TCP 5985 - Windows Remote Management (HTTP) - Allow TCP 5986 - Windows Remote Management (HTTPS) - Allow

Step 3: Link GPO to Target OU

Target OU:

OU=Workstations,OU=NEOS CIB 64,OU=SE,OU=CRDF,DC=euro,DC=net,DC=intra

Step 4: Force GPO Update

# PowerShell command to force GPO update on all GOT* computers: Invoke-GPUpdate -Computer "GOT*" -Force -RandomDelayInMinutes 0

Step 5: Test WinRM Connectivity

# On a GOT* machine, verify WinRM is running: winrm quickconfig # From management server, test connectivity: Test-WSMan -ComputerName GOT001

Alternative Quick Fixes

Option A: Run as Domain Admin

Launch PowerShell as a Domain Admin account before running the audit script.

Pros: No configuration needed

Cons: Security risk, may still fail on locked-down machines

Option B: Add Account to Local Administrators

# PowerShell one-liner to add scan account to all GOT* machines: $GOTComputers = Get-ADComputer -Filter {Name -like "GOT*"} -SearchBase "OU=Workstations,OU=NEOS CIB 64,OU=SE,OU=CRDF,DC=euro,DC=net,DC=intra" foreach ($comp in $GOTComputers) { Invoke-Command -ComputerName $comp.Name -ScriptBlock { net localgroup Administrators /add "EURO\YourScanAccount" } }

Pros: Persistent permissions

Cons: Security risk, manual cleanup needed

Summary

Recommendation

Use Group Policy (Option 1) as it provides:

  • Enhanced security (no over-privileged accounts)
  • Full audit trail
  • Easy reversal if needed
